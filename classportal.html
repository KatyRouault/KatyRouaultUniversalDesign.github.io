Labs

Lab 2 - Coding & Posting a Web Page - The Basics

This project involved recreating a multi-page Wikipedia article using semantic HTML5 structure, with sections organized through header, navigation, section, article, and footer elements. Anchor links provided smooth in-page navigation, while images were downloaded, optimized, and properly attributed. CSS was applied to enhance readability, hide elements visually but keep them accessible to screen readers, and maintain an optimal line length. The design used two Google Fonts (serif and sans-serif) and a color-blind–safe palette from Adobe Color. The final page was validated and tested for structure, contrast, and accessibility using HTML5 Outliner, W3C Validator, WAVE, and Silktide.Project Link

Lab 3 - Tables, Forms, Tabbed Browsing, and Screen Reading

This project involved creating a fully structured and accessible HTML5 webpage using real, meaningful content (“lifted content” rather than filler text). The page incorporated specific semantic HTML5 elements, an accessible form, and an accessible data table. It also included a CSS-based skip navigation dropdown for improved keyboard usability, Google Fonts for typography, and Google Material Icons hidden from screen readers. The design emphasized legibility and accessibility, validated through HTML Validator, WAVE, Silktide, HTML Outliner, and Lighthouse. As part of the project, screen readers (VoiceOver or NVDA) were used to navigate and evaluate accessibility firsthand, providing insight into user experiences for those relying on assistive technologies. Lab 4 - Accessible CSS Layout

This project focused on building a responsive, multi-column webpage structured with header, navigation, main content, two asides, and footer. The overall two-dimensional layout was created using CSS Grid, while the navigation links were arranged with Flexbox for a one-dimensional layout. Viewport settings and media queries were applied to optimize the layout for tablet and mobile devices, including reordering elements and converting the navigation from a horizontal layout to a vertical, thumb-friendly format on mobile. The design incorporated two Google Fonts, a color palette from Adobe Color, and accessibility-focused adjustments. The final page was validated using the W3C HTML Validator, HTML Outliner, and WAVE, and was tested with a screen reader to ensure accessibility before being published via GitHub Pages. Project Link

Lab 5 - Universal Design Website

This project centered on creating an informative, multi-page website that introduces and supports universal design for a new generation of web designers. The site was organized into dedicated sections explaining the purpose of universal design, its core principles, and the technical methods used to implement it. Built with a mobile-first, responsive approach, the design prioritizes usability, readability, and inclusive interaction across screen sizes. Accessible coding practices, semantic HTML, and thoughtful visual design choices were used throughout, and the site was evaluated using automated validation tools and screen reader testing to ensure it meets accessibility standards. The final website was deployed using GitHub Pages and presented as an accessible digital resource. Project Link

More about this class

This course explores the principles, standards, and techniques for creating websites that are accessible, inclusive, and effective across a wide range of users and devices. Emphasizing Universal Design, Web Standards, and accessibility best practices, the class covers topics such as semantic HTML, structured web design, user-centered design, mobile accessibility, and validation tools. Students engage with readings, expert resources, and video lectures to understand how cognitive and design theories impact usability and access. By the end of the course, students will be able to evaluate, code, and design web content that meets accessibility guidelines, supports diverse user needs, and adheres to modern web development standards.
